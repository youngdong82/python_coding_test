------------------------------------------  MySQL query 캐시
쿼리 캐시는 다른 DBMS에 없는 MySQL의 독특한 기능이다.
적절히 사용한다면 상당한 성능효과를 기대할 수 있다.

쿼리 캐시는 SQL문장을 캐시하는 것이 아니라 쿼리의 결과를 메모리에 캐시해 두는 기능.
쿼리 캐시의 구조는 key와 value의 쌍으로 관리되는 Map의 구조.
key는 query 문장 자체, value는 query의 실행 결과.

당연하지만, 동일한 쿼리가 요청되었다고 캐시에 저장된 결과를 보내면 안된다.
확인 절차가 필요하다.
1. 요청된 query 문장이 쿼리 캐시에 존재하는가?
비교대상에는 공백, 탭, 대소문자, 개행문자 등 전부이다. 따라서 쿼리 캐시를 위한 query는 문자열로 관리하는 것이 좋다.
2. 해당 사용자가 그 데이터에 접근할 권한이 있는가?
3. 트랜젝션 내에서 실행된 query인 경우, 가시범위 내의 트랜젝션에서 만들어진 결과인가?
4. query에 의해 만들어진 결과가 캐시하기에 너무 크지 않은가?
5. 캐시가 만들어진 이후 해당 데이터가 변경되었는가?
6. 쿼리에 의해 만들어진 결과가 캐시하기에 적당한가?
7. 그 밖에 캐시를 사용하지 못하게 하는 요소가 있는가?
 - 임시 테이블에 대한 query
 - 사용자 변수의 사용
 - 독립적인 SELECT 문장이 아닌 일부분의 서브쿼리
